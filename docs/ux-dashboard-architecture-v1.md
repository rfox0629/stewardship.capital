# UX Dashboard Architecture v1

## Purpose

The dashboard is the core product experience for Stewardship Capital.

The dashboard is the product. The PDF is a deliverable generated from dashboard data.

The MVP dashboard should give users immediate clarity after a fast assessment, then guide them into Financial Analysis for deeper opportunity discovery.

## Journey Context

Core product journey:

Landing Page -> Create Account -> 5 to 7 minute assessment -> Dashboard Created -> Explore Dashboard -> Complete Financial Analysis -> Unlock Opportunities -> Generate Professional Reports

## Dashboard Tabs

The dashboard uses six tabs:

- Overview
- Protect
- Grow
- Transfer
- Impact
- Reports

## MVP Dashboard Includes

- Stewardship Score
- Stewardship Stage
- Profile Completion %
- Protect / Grow / Transfer / Impact snapshot
- Top 3 priorities
- Next recommended step
- Locked future modules
- CTA to Complete Financial Analysis

## Overview Tab

The Overview tab should be the user's command center.

Required areas:

- Score and stage summary
- Profile completion
- Top 3 priorities
- Next recommended step
- Four-pillar snapshot
- Locked module preview
- Complete Financial Analysis CTA

## Pillar Tabs

Each pillar tab should provide a focused view of one area of stewardship.

### Protect

Guiding line: Protect what God has entrusted

Focus areas:

- Risk awareness
- Insurance review prompts
- Estate basics
- Emergency preparedness
- Document organization
- Professional team discussion topics

### Grow

Guiding line: Grow your capacity to serve

Focus areas:

- Business, practice, career, or investment growth clarity
- Real estate ownership context
- Tax-aware planning prompts
- Advisor coordination
- Opportunity readiness

### Transfer

Guiding line: Transfer values and resources to future generations

Focus areas:

- Estate plan awareness
- Family values conversations
- Succession planning
- Next-generation preparation
- Legacy communication

### Impact

Guiding line: Impact Kingdom and community

Focus areas:

- Giving strategy
- Community impact
- Kingdom priorities
- Charitable planning conversations
- Long-term impact intent

## Reports Tab

Reports should be generated from dashboard and analysis data. The Reports tab should not imply that a polished professional report exists before enough data is available.

MVP Reports tab:

- Preview locked professional reports
- Explain that reports unlock after Financial Analysis
- CTA to Complete Financial Analysis

Potential future reports:

- Stewardship Snapshot
- Financial Readiness
- Legacy Impact
- Opportunity Summary
- Professional Team Discussion Guide
- Family Stewardship Summary

## Key UX Principles

- Mobile first
- Desktop responsive
- Apple-level simplicity
- Premium Christian family office feel, but not family-office-only
- Avoid cluttered fintech dashboard feel
- Use plain language, not advisor jargon
- Prioritize clarity over density
- Make the next best step obvious

## Assessment UX Requirements

Assessment v1 should include:

- 25 questions max
- 5 to 7 minute completion target
- Mostly Yes / No / Not Sure and multiple choice
- No dollar amounts
- No account connections
- No uploads
- Auto save shown at top
- Resume later is required

The assessment header should make saving visible so users never fear losing progress.

## Progressive Onboarding

Never force users through a giant intake form.

The dashboard should progressively unlock deeper detail:

1. Assessment creates initial dashboard.
2. Dashboard reveals stage, scores, pillars, and priorities.
3. User completes Financial Analysis.
4. More opportunities and reports unlock.
5. Future modules deepen the operating system.

## Recommendation UX

Recommendations should:

- Map to Protect, Grow, Transfer, or Impact.
- Use plain language.
- Avoid direct legal, tax, or investment advice.
- Be framed as topics worth discussing with the user's professional team.
- Create a clear path to Financial Analysis when more information is needed.

## Locked Future Modules

Locked modules may include:

- Financial Analysis
- Opportunity Engine
- Professional Reports
- Professional Network
- Estate and Legacy Vault
- Giving Strategy
- Business Owner Planning
- Real Estate Owner Planning
- Advisor Collaboration

Locked modules should create anticipation without frustrating users.
